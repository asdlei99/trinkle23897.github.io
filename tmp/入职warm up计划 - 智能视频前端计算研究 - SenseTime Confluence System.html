    入职warm up计划

    Skip to end of metadata
    • Created by Qin Hongwei, last modified by Xu Liang yesterday at 3:09 PM
    Go to start of metadata

    各位， 我们经常有新员工入职。入职warm up是一个重要环节，之前我们的做法是直接进入项目。团队还很年轻，我们需要不断探索最合适的方式。

    从2018Q1开始，我们的入职warm up修改为论文阅读讨论方式：

    Paper List：

    按照顺序读

    1. CNN 分类 (LeNet, AlexNet, ZF-net, VGG, GoogLenet v1 v2 v3, ResNet, GoogLenet v4, MobileNet, MobileNet v2)

    2. Detection（R-CNN， overfeat，SPPNet， Fast R-CNN, Faster R-CNN, R-FCN, FPN, Mask R-CNN, Focal-loss)

    3. Tracking（GoTurn， Siamese FC）

    4. Video Object Detection （DFF， FGFA， Impression Network，Detect to Track and Track to Detect）

    5. Model Compression

    > Deep Compression

    > [Distilling the Knowledge in a Neural Network](https://arxiv.org/pdf/1503.02531)
    > [FitNets: Hints for Thin Deep Nets](https://arxiv.org/pdf/1412.6550.pdf)
    > [Mimicking Very Efficient Network for Object Detection](http://openaccess.thecvf.com/content_cvpr_2017/papers/Li_Mimicking_Very_Efficient_CVPR_2017_paper.pdf)
    近期入职的同学：迟程、孙杨威、徐亮、李润东
    徐亮目前基本上完成前两部分论文的阅读，之前采用的方式是每读完几篇之后，给我讲解一遍。但这种1v1的沟通，需要占用的时间比较大，共性问题也会比较多。所以，后续我们采用小组共同学习的方式。具体做法：
    1. 请大家在本blog下填空，可以是附件的形式，或者编辑正文的形式，填写自己的阅读笔记，挑重点（比如徐亮可以记下来给我讲解的过程中我提的那些问题）
    2. 之后论文讲解，还是一对一的方式，但一定注意不要出前一位同学的共性问题
    3. 大家有建议的资料、论文，可以加进来，不断完善

    以上，请今日起，> Xu Liang 分享笔记 > Li Rundong 跟进。

    在完成warm up的过程中（通常应为1周左右），请务必多与同组其他同学交流研究内容，以便之后确定研究方向。

     

    第一次论文阅读笔记分享（2018.3.1，参与人：秦红伟，徐亮）

    分享内容：LeNet, AlexNet, ZF-net, VGG, GoogLenet v1

    • LeNet
      • 1：从传统的分类框架跳跃到使用神经网络来实现分类任务，指出传统的方法最大的缺陷是没有实现learning from data
      • 2：由于是第一个吃螃蟹的，所以对神经网络的每个模块进行了分析和含义的分析，详细的分析了卷积层/池化层/全连接层/激活函数等目前来看是神经网络标配的模块的含义
      • 3：网络结构
    • AlexNet
      • 1：首次在CNN使用ReLU激活函数并取得了很好的效果（先分析Sigmoid函数在BP过程中存在的先天不足，然后引出ReLU的优势）
      • 2：提出防止过拟合的很多实用的训练trick：random crop，flip，dropout等
    • ZF Net
      •  1：大致结构和AlexNet相当，只是在filter size这些小地方做了一些小的调整（红博士指出：理解filter size改变的原理和意义）(补充：网络结构和AlexNet很相似，其中第一层卷积由11*11改为了7*7，理由是较小的卷积能保留原始的pixel信息，而较大的卷积会skip掉较多的信息，特别是对于第一层直接连接image的卷积)
      •  2：主要贡献在于通过解卷积的思路来帮助我们可视化并且进一步理解CNN内部，实现解卷积的核心思路也很简单清晰，是文章很有价值的亮点
    • VGG
      • 1：最大的贡献在于保持网络结构简单清晰的前提下，把CNN做的更深了，更契合deep learning中的deep的含义，在此之前的AlexNet只有五层，VGG一下子做到了16层，19层
      • 2：讨论了使用不用的卷积核大小的优劣势，比如使用3*3比7*7各自的优劣势是什么？答案是7*7的感受野是更大，但是参数太多，3个3*3的卷积核能达到和一个7*7同样的感受野，但是二者的参数量分别是3*3*3=27 VS 7*7=49，因此使用小的卷积核能够很好的减少参数，那么更进一步使用1*1是不是也会比3*3更好呢？以及为什么卷积核是奇数呢？为什么不是2*2或者4*4的卷积核呢，都可以看这篇文章，并加入自己的思考得出答案
      • 3：由于VGG是第一个把网络变得这么深的，而当时大家面对这么深的网络并没有一下找到很好的train方法，因此VGG的train网络部分需要精读并理解（红博士指出：理解不同的训练方法发展阶段，如xavier初始化方法发明之前和之后的网络训练差异）
    • GoogLenet v1  
      • 1：提出Inception模块卷积，通过结合多个尺度的卷积核的优势来增强模块学到的东西
      • 2：理解Inception模块的结构的含义，以后这样设计会存在哪些问题，如因为Inception模块是在每层的输出在channel上串起来得到的，如果不做处理的话这个channel会增加到很恐怖的尺度，然后你直接用3*3或者5*5的卷积核来处理这么多channel的话你的参数会爆炸，根本不可行，这就是图中每个模块中3*3和5*5之前加一个1*1的卷积核的作用（也就是在channel上降维，减少参数量）。
      • 3：通过叠加多个Inception模块,能够实现网络的deep，这里面针对训练 deep 网络提出了另一种思路，就是在网络的中间层去引出辅助训练的loos，增强监督信息，这种思路也很值得学习。
      • 4：文中引用的另一篇文章NIN（NIN-Network In Network）思路（红博士指出：对文中引用的一些有价值的文章也应该参考阅读，把控细节）

    1st END

     

    第二次论文阅读笔记分享（时间地点待定）

    分享内容： GoogLenet v2 v3, ResNet, GoogLenet v4, MobileNet, MobileNet v2

    • GoogLenet v2
      • 主要网络框架依然是沿用Inception模块，但是和主流一样，都转向小的卷积核，因此原Inception模块中的5*5卷积核被两个3*3的卷积核替代了，替换的理由和VGG将7*7换成3*3一样，即是用更少的参数，而且能得到更多得的非线性，一举两得
      • 另一个重要的工作就是提出使用BN（Batch Normalization）来加速CNN的训练：
        • 随着网络的加深，训练很深的网络是比较困难的， 因为网络中的feature的尺度不一样，每层网络的输入的数据分布也不一样，因此特别是对于S函数这种在极大值和极小值的两端梯度都会消失激活函数，输入scale带来的问题是很严峻的，很可能导致网络中后面某些层的梯度一直是0从而无法很好学习
        • BN的作用：BN通过将每层的input进行归一化，使得每层接收的输入都服从均值为0，方差为1。符合这样分布的输入数据在BP的时候能够保证不管网络多深，在任意层输入的scale都是一致的，从而使得BP更有效
        • 如何实现BN:在每个mini-batch里面计算当前batch内部数据的均值和方差，然后对输入数据进行归一化（归一化是基于channel的，每个channel内部进行归一化），同时为了保持网络特征的参数的表达性和可视化意义就会随之被破坏了，因此需要对归一化后的输出x再学习一个scale A和shift B，y=Ax+B，然后将y经过激活函数以后输出到下一层
      • 使用BN以后带来的好处
        • 1：网络学的更快更好了，可以将learning reate设大一点，加速训练
        • 2：有了BN可以减少dropout，去掉LRN
    • GoogLenet v3
      • 最核心的思路：Network in Network，把Inception模块中的n*n卷积全部替换成1*n+n*1的卷积，减少参数量，文中通过实验还得出经验性的结论：对于中等大小的feature map（如20*20），使用1*7+7*1的卷积核效果是最好的
      • 把Inception模块进一步扩展，横向和纵向都加深加宽，如下图：
    • ResNet
      • 1：同样关注的是网络变深以后难以训练的问题，因为网络太深以后后面的很多层在BP的时候由于链式求导法则的存在，难以有效的更新网络参数
      • 2：通过设计残差网络来使得网络进一步加深的同时，梯度能够更有效的传导下去
      • 3：典型的残差模块如下：
        • ，
        • 在输出层输出的不是当前网络学到的映射函数F(x)，而是x+F(x)，如果F(x)和x的channel不统一的话就用1*1的卷积核来做channel的对齐
           4：在每个残差模块内部为了减少参数量，也先经过1*1的卷积核进行channel降维，然后用3*3做卷积，其后再用1*1来做channel的升维

      • 1：主要接收了ResNet的优点，将残差模块和Inception模块结合起来

    • MobileNet
      • 1：研究重点是把经典的卷积核在空域和channel维度上进行解耦，前面的depthwise卷积在每个channel上单独做，做完以后再用1*1的卷积核做pointwise卷积，这样能很大程度上减少卷积需要的参数量
      • 2：能多大程度上减少卷积需要的参数：以输入feature map为3*3*100，输出为10，使用3*3的卷积核来计算：
        • 传统卷积计算量：3*3*100*10=9000

      • 3：减少参数以后对性能的影响：
        • ，由图可见MobileNet性能可以做到和VGG16相当，但是参数减少32倍，性能比GoogLenet 更好
    • MobileNet V2
      • 1：由于MobileNet v1关注的主要是讲depthwise卷积这个思路引进来，因此它在神经网络的设计上比较简单粗糙，和VGG类似的直筒型网络结构，因此MobileNet V2自然而然的将网络结构进行了改进，将ResNet思路和depthwise卷积结合起来
      • 2：Linear Bottlenecks ： 作者通过实验论证发现对于不同channel数目的feature map经过ReLU以后再信号还原的时候，channel数目少的在还原的时候受到的影响更大，如下图：
      • 因此对于网络中用于channel降维的层后面都不再使用ReLu处理，而是使其保持线性
      • 3：Inverted residuals ：之前的 ResNet在每个残差模块内部为了减少参数量，要先经过1*1的卷积核进行channel降维，然后用3*3做卷积，其后再用1*1来做channel的升维，我们这里因为用了depthwise卷积卷积来减少参数，因此ResNet的残差模块被修改为先用1*1的卷积核进行channel的升维，来进行特征的丰富化，然后使用depthwise卷积处理，再使用1*1的卷积进行channel的特征降维

    第三次论文阅读笔记分享（时间地点待定）

    分享内容：Tracking（GoTurn， Siamese FC）

    • 1:GoTurn
      • 1:以提升检测速度为出发点的检测框架，检测最快速度可达到165FPS，能够做到这么快的主要原因：1：网络的训练是offline的，在跟踪的时候不需要进行参数的update；2：很多方法测试时筛选出很多可能的image patches ，然后选出最优的检测目标，而本文只需要将可能的一个region输入网络进行前向传播，就能回归得到正确的目标位置，极大的加速。
      • 2：算法框架：
        • ，
        • 网络接受的输入是前一帧和当前帧，在前一帧的找到我们需要track的目标框，在当前帧中取出搜索域中的那部分，然后将两个图片输入到在ImageNet数据库上预训练好的CaffeNet的前五个卷积层（这五个层的参数冻结，不参与BP），取出pool5 features，然后将两个串起来当做一个feature输入到后面来的全连接层中，最后一个全连接层的输出是4维的向量，代表的含义是tracking的目标的位置（x,y,w,h）
        3：网络的输入：
        • 1：对于t-1帧，提供的是目标的groundtruth bounding box信息，假设目标所在位置为（cx,cy），其大小为（w,h），在目标中心处提取一块大小为（2w,2h）的图像块输入到CNN中，宽度高度分别变为两倍的依据就是这样能够包含context信息
        • 2：对于在第t帧，也以（cx,cy）为中心，提取大小为（2w,2h）的图像块，输入到CNN中，可以直接以前一帧的中心位置作为当前帧的目标位置的理论指导是对于相邻帧，目标的位置变化一般不会过大，相应的对于移动速度很快的情况下，需要将context的范围扩大，或者使用online-trained object detector来进行改进，本文目前没有对此进行更深入的讨论
      • 4：如何训练这样的网络：
        • 1：从video中获得训练样本：随机选取一个video，然后从中随机选取相邻两帧
        • 2：从still image中获取训练样本：我们通过对一张图片进行特定的motion smoothness得到变化后的图片，然后将二者组成训练样本，我们可以将二者视为是来自视频中的不同帧，虽然这两张图片之间的运动变化不如取自真正的video中的多，但是对于帮助网络学到更多的不同类型的目标还是有帮助
        • 3：motion smoothness：作者通过对训练集中相邻帧目标运动的建模分析，发现当前帧的目标的位置和尺度的变化和前一帧是存在某种分布关系的，数学公式此处不做推导，结论就是对于x,y位置的变化，符合均值为0，尺度为1/5的laplace分布，对于w,h宽度高度的变化符合均值为1，尺度为1/15的laplace分布。因此在处理still image时可以将原图经过符合上述分布的变换，得到对应的新图。具体方法就是对原图进行符合上述分布的random crop，同时为了防止random crop后框中目标太少，对变化过程加以限制：宽度和高度的变化尺度范围控制在(0:6; 1:4)，同时保证random crop时长和宽两个尺度上的原目标50%以上的区域要被框选中。motion smoothness示例图片：

        • 4：data augmentation：data augmentation对于图片目标和视频目标都在训练中使用。在训练过程中，我们使用和motion smoothness思路相同的random crop方法对每个目标crop出10倍的目标进行数据增

      • 5：对比实验

        • 通过对比实验得出更好的loss设计，以及同时使用video训练数据+image训练数据的好处：通过从视频中进行训练学到了对于运动，旋转，光照和尺度的不变性；通过从still image中学习能够学到多种不同的目标的特征，提高模型的适应性。

                  蓝色曲线代表测试样本的同类型在训练集中很少出现，绿色代表训练和测试样本有较多的同类样本，实验表明：
        •     1：即使训练样本和测试样本相同类别的数据较少或者没有，网络的表现还是不错的，因此具有较好的普适性，在一些不可控的检测环境中表现也不会很差；
        •     2：通过增加同类的训练样本出现频率，性能得到明显提升，说明将网络应用在不同的特定任务的时候，只需要增大该任务特有的训练样本即可
      • 6：框架的优点：
        • 1： 训练都是基于offline的，不需要online进行参数更新，速度快
        • 2：预测时网络只需要进行一次forwad就能输出正确的边框坐标，不需要产生很多个可能的proposal区域然后再筛选的过程，进一步加
    • 2:SiameseFC
      • 1:SiameseFC的算法流程图:

        • 1: 上面一个支流：z是第一帧目标给出的需要track的目标的框（这里也叫模板），对其使用一种特征提取器φ（φ可以是任意形式，这里使用的是含有五个卷积层的CNN网络），得到的输出是φ（z）（这里是conv5的输出也就是6*6*128的feature map）

          2：下面一个支流：x是当前帧的搜索区域，同样经过特征提取器φ以后得到的输出是22*22*128的feature map，记为φ(x)

          3：上述两个feature map经过一个 * 互相关层联合起来得到17*17*1的最后feature map的输出。可以看成是φ(z)在φ(x)上滑动搜索（和每个滑动的位置计算内积），最后得到一个响应图，图上最大值对应的点就是和当前的搜索模板匹配的最好的点（也就是算法认为的目标中心所在位置）

          4：通过mini-batch一次可以对同一个模板和同一个目标进行多个尺度的模板匹配搜索

      • 2：如何训练网络：
        • 1： loss函数：，y=+1,-1；当v较大且y=1时，认为跟踪正确，此时损失函数的值很小，当v较大且y=-1时，表示跟踪到了错误的位置，此时损失函数的值很大
        • 2：训练阶段score map上正负label的指定：每个在score map上的元素距离groundtruth的中心半径小于R的指定为正样本，其余为负样本
      • 3：算法性能：

                      可以看到本文的算法在精度能够超越大多数模型的同时，还能够保持很快的速度，足以证明这个框架是简洁有效的。而且只需要通过度量学习的方法就能实现，比边界回归之类的方法更简洁

    第四次论文阅读笔记分享（时间地点待定）

    分享内容：Video Object Detection （DFF， FGFA， Impression Network，Detect to Track and Track to Detect）

    • 1：DFF：Deep Feature Flow for Video Recognition
      • 1：解决的主要问题是传统的video recognition需要将视频按帧进行提取深度特征，这是很耗时的步骤，而本文deep feature flow只需要对视频中稀疏的几个关键帧进行深度特征提取，在其他位置只需要通过deep feature flow来进行特征推导得到特征， 这就能免去在每一处都提取CNN特征的耗时。能使用关键帧的理论依据是视频的帧都是连续的，相邻帧不仅在RGB域上看起来很像，在卷积后得到的feature map上更像（Consecutive video frames are highly similar. The similarity is even stronger in the deep feature maps ），因此可以通过本文的方法来进行非关键帧位置的特征推导
      • 2：框架：
        • DFF的框架最大的不同就是对于当前帧没有使用特征提取器Nfeat直接提取其特征， 而是根据关键帧信息和本文的DFF来进行特征推导，得到当前帧的特征，通过精心设计使得DFF推导的结果比较精准的同时速度还很快。算法框架的伪代码：
        • 可见算法的最重要的逻辑是对于当前帧是关键帧和不是关键帧的不同处理方式：
          • 当前帧是关键帧的时候，使用特征提取器Nfeat直接提取其特征，并把当前帧设为随后几帧的关键帧
          • 当前帧不是关键帧的时候，使用之前的关键帧和当前帧来进行特征流的传播，来得到当前帧的特征
        • 上述伪代码中几个关键模块各部分的意义
          • Nfeat：特征提取模块，这里使用的是ResNet-50和ResNet-101的conv5层的输出
          • Ntask：子任务模块，可以在feature map后面接入不同的网络结构完成不同的任务，如本文试验了可以完成语义分割任务和目标检测任务
          • W（fk，F（Ik，Ii），S（Ik，Ii）)：F（Ik，Ii）指的是对这两帧计算其光流信息（文中使用的是FlowNet 网络），得到光流信息以后将光流的feature map大小调整为和特征提取模块得到的feature map一样大，然后将这两个feature按照一定的规则进行运算得到新的feature map，S（Ik，Ii）)指的就是对这个新的feature map加上一个尺度变换，W()就是将这几步操作组合起来的一个示意代名词。

      • 3：框架中的其他点：

        • 1： 论文中多次提及的关键帧选取策略：论文使用的是固定步长选取设定为关键帧的方法，主要原因是简单，作者也提到更好的关键帧选取方法应该会进一步改善本文的性能，但在本文中不予讨论

        • 2：本文设计的框架快在哪里：主要差异在对于非关键帧的处理上，Nfeat使用的是ResNet-50或者ResNet-101这样复杂的网络来提特征，而作者先使用FlowNet 网络来估算光流，然后后面的运算都是很简单的逻辑运算，FlowNet 网络本身又很简单，层数也不多（不到10层），因此总体还是快很多，尤其是非关键帧的数目本身也比关键帧多很多

      • 4：算法性能：
        • 表格里面呈现的是语义分割任务和object detection 任务：
          • Cityscapes数据库+semantic segmentation任务： 和按帧提取特征的性能有差距，说明通过deep feature flow进行推导得到的特征还不够好，但是已经比手工构造的SIFT-Flow 特征好了，同时因为设定的固定每5帧取一个关键帧，对应的FPS提升了3.7倍，可见本文的框架在速度和进度之间的平衡做得很好
          • ImageNet VID数据库+object detection 任务： 73.9%的结果使用了很多的trick(multi-scaletraining/testing, adding context information, model ensemble)，而本文没有使用这些tricks依然得到了最好的73.1% ，同时设定的是每10帧取一个关键帧，对应的FPS提升了5倍，说明本文的方法性能稳定性也很好
    • 2：FGFA：Flow-Guided Feature Aggregation for Video Object Detection
      • 1：算法框架：
        • 对于检测视频中的目标面临的主要困难是物体的运动导致的模糊，角度以及目标的对焦问题。本文通过融合视频帧前后相邻位置的特征，使得到的特征对于运动目标的检测更具有鲁棒性，因为此时一帧的特征不仅仅和他自身的特征相关，还通过计算相邻位置的光流信息，然后以光流信息为媒介，将前后的帧的特征融合到本帧当中来，简而言之就是单帧特征变好了
      • 2：主要模块：整体而言，对于当前帧t，网络会去观察他前后K帧，也就是从t-K到t+K范围内的帧，然后将这些信息和当前帧的特征进行融合。因此大体分为Flow-guided warping 和Feature aggregation 和Adaptive weight三大模块
        • 1：Flow-guided warping ：

    • 3：Impression Network for Video Object Detection
      • 1：算法框图：
        • 1：整体框图和DFF是差不多的，还是使用Nfeat来提取关键帧特征，而且本文的关键帧选取也是在固定长度的一段里面选一帧为关键帧，然后对于两个关键帧之间的非关键帧的区域依然使用和DFF一样的特征推导方法
        • 2：区别在于为了得到ftask这个参数（DFF中等同于fi），DFF使用的是，而在Impression Network for Video Object Detection中使用的是，因此这里得到的当前的ftask不仅仅和当前的帧的特征相关，还和新引入的参数fimp相关，而fimp又会在后面随着网络不断更新，fimp包含的信息不仅含有当前帧的信息，同时含有一定的全局的时域信息，因此这样更新得到的特征会更好。实验结论也证明了这一点：
        • 3：和FGFA相比：
          • 由于FGFA的特征更新需要和当前帧前后的K帧的特征进行加权融合，因此速度较慢，但是融合后的特征对于运行模糊的鲁棒性更好，所以本文的方法速度更快，但精度稍弱于FGFA
      • 本文的创新点正如abstrac中所说的一样，通过关键帧的特征推导能够极大的加速；通过融合相邻多帧的信息能够得到更好的特征，但是速度会变慢；因此本文是想在获得加速的同时把全局信息或者相邻帧信息融合进来，是在DFF和FGFA之间的一个平衡点。主要思想和细节将这三篇文章连起来读会更好，在此细节不再展开
    • 4：Detect to Track and Track to Detect
      • 1：算法框架
        • 通过一个CNN网络，同时实现检测和跟踪两个任务
        • 对于两帧图片t和t+τ，分别用CNN提取特征，紧随着后面网络横向的流动的思路和R-FCN是一样的，对最后一个CNN的feature map上做RPN生成ROIs，然后对position-sensitive score上做RoI pooling，对RoI pooling得到的结果分别接入实现分类和坐标回归的fc结构
        • ROI Tracking：通过一个Correlation操作将t和t+τ两帧的feature map关联起来得到新的correlation features ，并基于correlation features 可以用于track的loss，从而通过坐标回归实现track的目标
      • 2：主要模块：
        • 用于detection的基于单帧的回归和分类两个模块：和R-FCN思路一样，CNN端使用ResNet来提特征，然后根据需要对conv5的stride进行修改（这里对conv5的处理细节和R-FCN不同），最后得到的用于分类的position-sensitive score maps 也是k2(C + 1)  ，同理用于回归的大小是4k2 ，文中使用的k × k = 7 × 7 ，R-FCN中标准的k × k = 3 × 3，后面的回归和分类loss的设计也和R-FCN一样，包括训练中这两个任务的正负样本指定也是一样的
        • 用于tracking的模块：
          • 综合三个任务的loss：，文中取的λ为1
          • 用于tracking的loss-Ltra：是tracking回归的目标，当t和t+τ两帧都出现同一目标时，回归目标就是t+τ帧中 ground truth targets，形式依然是坐标形式（x,y,w,h），回归目标的规范化依然和R-FCN中的坐标目标规范化一样
        • Correlation features ：
          • 如何得到：见论文3.4. Correlation features for object tracking，不再展开
        • Linking tracklets to object tubes ：
          • 将参数t+τ中的τ从帧扩展到tube级别，能够减少运算量，具体描述见论文4. Linking tracklets to object tubes
      • 4：算法性能：
        • 结论：
          • 1：D（R-FCN）：从本文的框架中去掉用于tracking的loss-Ltra，由于RPN中用的是15个anchors，比之前的9个anchors性能有所提升（74.2% vs 73.9%）
          • 2：D（& T loss）：在1的基础上把用于tracking的loss-Ltra加上，可见结果有提升，证明本文设计的loss是有效的（75.8% vs 74.2% ）
          • 3：D&T（τ=1）：在测试的时候τ=1代表每帧都进行track，在最佳性能上超过state-of-the-art，79.8% vs 76.19%
          • 4：D&T（τ=10）：在测试的时候τ=10代表每10帧进行track，然后使用第四节的方法将中间没有进行检测的帧进行link起来，这样能减少大约10倍的时间，但是性能仅下降了1.2%
